menu

_g.menu was created to allow the developer to easily create customized context menus on elements. to do this, the script must first override the default context menus set by the browser. this may be convenient as it sets a standard design across all browsers. however, this feature may not be for everyone and it is recommended that you look elsewhere if this behavior is not suitable for your situation. the script was originally designed for Graphene and therefore uses the conventions for related _g scripts.

getting started

to begin using the menu script, one must first include the library in the html file. the source file can be downloaded here and put on a cdn the inclusion of this script will immediately begin to override the browser's context menu without further implementation.

basic features of the default menu

the basic menu, when not targeting a particular element, includes navigation options: Back, Reload, Forward, and Scroll to top. this last option only shows when the user has scrolled down the page beyond one screen height, and implements a smooth scroll script to scroll to the top of the page. it will also contain a link to the page source: View Source.

when text is selected (you may test this by selecting this text) the menu will show options to Copy the selected text or Search Google for the text. the menu when a link is targeted will show the options: Open Link in New Tab, Copy Link Address, and Save Link As..., you may test this with this link: Google.

if the user targets a video when right clicking, the menu will show options to control the playback of the video: Play/Pause, Mute; it will also show options to Copy Video Source or Save Video As... here is a video on which you can test these features:

adding callback functions

to add more fluidity between the context menu and the page, callback functions can be added to both _g.open() and _g.close(). for _g.open(), the callback function is stored as _g.m.onOpen(), and for _g.close() it is _g.m.onClose(). as an example, the content of the page can be blurred using filter:blur when opening the context menu and unblur when closing. this can be seen by toggling this example ON and OFF.
